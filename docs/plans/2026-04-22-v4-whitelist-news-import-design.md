# V4 白名单新闻源导入设计稿

## 目标
把当前 live console 的入口从“手动粘贴全文”升级成“直接粘贴新闻链接”，同时保持现有真实支付和 Arc receipt 主链路不变。页面输入链接后，系统自动导入正文，执行一次按次付费的 agent session，并输出轻量图谱。

## 推荐方案
采用“白名单新闻源导入 + 轻量关键词图谱”的保守方案：
- 只支持 `wublock123`、`PANews`、`ChainCatcher`
- 使用原生 `fetch` 和站点专用 extractor
- 不做 headless 浏览器，也不做全网抓取
- 图谱输出收敛到 `2-4` 个节点、最多 `3` 条边
- 对强实体新闻保留实体导向输出，不做一刀切降级
- 固定抓取 guardrail：`8000ms timeout`、`1_000_000 bytes` 响应体上限、`3000` 字正文截断

## 交互方案
`/demo/live` 页面新增：
- 输入模式切换：`文章链接 / 手动文本`
- 白名单来源说明
- 带本地缓存导入结果的预置新闻卡片

推荐录屏路径：
1. 打开 `/demo/live`
2. 选择一张预置新闻卡片或粘贴白名单链接
3. 点击开始
4. 展示支付过程、关键词图谱和证据区
5. `latest/detail/polling` 和 graph 页面都能看到来源链接与来源站点

## 实现切片
1. `news-import` domain
2. 共享 source 契约与 live session `articleUrl` 模式
3. 轻量关键词图谱退化
4. graph 与 runbook 回填

## 风险
- 页面结构变动导致抽取器失效
- 低信息量文本图谱过空
- 链接导入失败影响录屏稳定性
- URL 契约不清导致 live API 分支复杂

## 控制
- 只做 3 个站点
- HTML fixture 测试锁定结构
- 页面保留带本地缓存导入结果的预置卡片兜底
- 图谱坚持“少而稳”，而非“多而乱”
