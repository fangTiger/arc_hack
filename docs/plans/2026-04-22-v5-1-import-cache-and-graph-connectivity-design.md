# V5.1 抓取缓存回退与图谱连通性设计稿

## 目标
这一轮只做“稳定性补丁”，不碰 UI 大改。目标是：
- 新闻链接第一次成功后可以缓存
- 同链接以后抓取失败时自动回退缓存
- 图谱尽量不再展示多个孤立节点

## 推荐方案
采用“实时抓取优先 + 同 URL 缓存回退 + 展示层连通性修正”的保守方案：
- 不引入 headless 浏览器
- 不做跨站搜索
- 不改变原始 `relations`
- 只修新闻导入和 graph 展示层
- 固定 URL 规范化规则，并用 request/finalUrl alias 命中同一缓存

## 交互结果
- live console 链接导入失败时，如有缓存仍可继续运行
- 页面明确标识“实时抓取 / 缓存回退”
- graph 页面优先显示连通结构，不让多个孤立节点直出
- graph 页面会说明“辅助边仅用于展示”

## 实现切片
1. 新闻导入缓存 store 与 URL 规范化
2. `importStatus/cachedAt` 元数据贯穿 live payload
3. graph 连通性后处理与 `derived` provenance
4. graph/live 文案与 runbook 回填

## 风险
- 缓存被误解为实时成功
- 自动补边被误解为真实关系

## 控制
- 明确展示缓存回退状态
- 辅助边只存在于 graph 展示层
- 原始抽取结果不改
