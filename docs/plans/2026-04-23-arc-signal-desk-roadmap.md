# Arc Signal Desk V0.2 路线与品牌方案

## 1. 产品定位

`/demo/live` 下一阶段不再继续强调“可信投研工作台”这种偏比赛答辩式命名，而是统一升级为：

**Arc Signal Desk**

它的职责不是展示模型会做什么，而是把一条资讯压缩成一页可执行判断，帮助用户在最短时间内完成：

- 理解发生了什么
- 判断值不值得跟进
- 核验依据是否站得住
- 追溯调用与支付凭证

一句话定位：

> 把资讯、证据与调用凭证收束成一页可执行结论。

## 2. 品牌语言系统

为了去掉“low”的功能堆砌感，页面语言统一从“工具描述”切换到“决策语言”。

建议替换如下：

- 产品主名：`可信投研工作台` -> `Arc Signal Desk`
- `导入仓` / `收纳式导入仓` -> `线索入口`
- `事件总览` -> `执行摘要`
- `关键判断` -> `核心结论`
- `证据摘录` -> `证据锚点`
- `延展阅读` -> `深度解读`
- `情报侧栏` -> `信号侧栏`
- `辅助关系图` -> `关系导航`
- `分析凭证` -> `调用凭证`
- `分析流程` -> `执行轨迹`

这套命名的目标不是“更花哨”，而是让每个区块都更像一个真实产品模块，而不是一个解释功能的临时标签。

## 3. V0.2 优先级

### P0：结果优先

分析完成后，页面应该明显进入“结果阅读模式”：

- 左侧 `线索入口` 保持可用，但视觉权重要降低
- 首屏只服务 `执行摘要 -> 核心结论 -> 最强证据`
- `关系导航`、`调用凭证`、`执行轨迹` 继续存在，但不抢走主区焦点

### P1：动作闭环

当前页面告诉用户“发生了什么”，下一步要补上“你现在可以做什么”：

- 复制摘要
- 导出简报
- 标记为待跟进
- 生成分享链接

### P2：历史回看

产品不能停留在一次性演示。下一阶段需要规划：

- 最近分析记录
- 同主题结果对比
- 不同来源交叉验证

### P3：品牌统一

live 页、graph 页、README 对外描述都要统一到同一套品牌语言，避免一处叫 `Desk`、一处叫 `工作台`、一处又回到 `demo console`。

## 4. 本轮实现边界

本轮先做最小但可感知的品牌升级：

- 更新 live 页主标题、分区标题和主题相关文案
- 更新 graph 页相关命名
- 更新对应测试锚点，保证 UI 文案与行为一致

暂不做：

- 新增后端接口
- 新增历史记录模型
- 大规模布局重排
- 新动作按钮实现

## 5. 验收标准

完成后应满足：

- 首屏不再出现 `可信投研工作台` 这类旧命名
- 页面语言从“工作台说明”转为“决策产品语言”
- live 和 graph 页面命名风格保持一致
- 现有测试更新后通过，界面功能不回退
