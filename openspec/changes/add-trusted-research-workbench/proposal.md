# 变更：add-trusted-research-workbench

## Why
当前 `live console` 虽已具备链接导入、阶段展示、支付证据、关键判断、证据摘录和辅助关系图等能力，但产品叙事仍然残留明显的 demo 壳层：

- 首屏仍存在 hero、输入区、结果区多焦点竞争，用户不知道先看什么
- `关键判断 -> 证据 -> 来源上下文` 的可追溯闭环还没有被定义成 P0 产品约束
- 页面状态与分析阶段混在一起，导致空白态、分析中、失败恢复、重新分析的产品语义不稳定
- 右侧侧栏承载了过多次级信息，容易重新滑回“信息堆叠页”而不是工作台
- 当前实现虽然完成了“产品模式化”第一轮结构收口，但页面整体观感仍明显延续旧版浅色演示页样式，首屏焦点与版式张力不足，用户仍会把它感知为 demo 的延伸
- 新版前端在真实浏览器中还暴露出一个关键稳定性问题：分析任务服务端已经完成，但工作台可能继续停留在“分析中”，说明轮询结果存在终态覆盖或过期快照渲染风险

这次改版已经不再以“比赛演示页”或“live console”思维推进，而是明确升级为 `资讯分析工作台产品模式`：

- 主价值是帮助用户在同一页面里快速建立判断、核验证据、复查来源与凭证
- 页面主叙事固定为 `事件总览 -> 关键判断 -> 证据摘录 -> 态势侧栏 -> 分析元信息`
- 关系图只能作为辅助态势视图，不能再承担主结论职责
- 重新分析与失败恢复必须保留工作台上下文，不能把用户打回一次性表单流程

因此需要在现有变更上补充产品模式化 proposal/design/spec，把这套工作台从“已具骨架的 demo 页”收敛为“可连续阅读、可立即复核、可低成本重跑”的产品界面。

在此基础上，本轮继续追加一轮 `结果优先的情报分析台` 强化重构，明确移除旧 demo 布局痕迹，并修复浏览器端终态卡住的问题。

## What Changes
- 将 `live console` 明确定义为 `资讯分析工作台产品模式`，移除用户侧 demo 叙事与展示页式 hero
- 固定页面信息架构为：顶部状态总控条、左侧收纳式导入仓、中间主结论区、右侧态势侧栏
- 固定主阅读链路为 `事件总览 -> 关键判断 -> 证据摘录`，并把 `判断-证据-来源上下文可追溯` 升级为 P0 约束
- 引入双层状态模型：区分用户可感知的页面状态与系统内部分析阶段，约束骨架屏、状态提示、按钮行为与异常恢复
- 明确 `首页默认空白` 与 `重新分析保留旧结果` 的产品规则，避免失败或重跑时整页回退空白
- 将来源、流程、支付与 receipt 证据统一收纳为分析元信息入口，避免侧栏碎片化
- 将视觉语言收敛为“冷静克制的专业未来感”，禁止页面再次滑回炫技 demo 或大屏看板
- 将首屏升级为 `顶部工具带 + 宽主阅读区 + 稳定右侧情报侧栏 + 右侧详情抽屉` 的结果优先产品布局，输入入口默认退为工具行为而非主视觉区
- 在轮询链路中增加终态优先与响应新鲜度保护，避免过期 `running` 快照覆盖 `completed/failed` 终态

## Out of Scope
- 通用 URL 解析或更多新闻站点支持
- 大规模图谱算法平台化、graph database 或复杂图谱编辑器
- 新增历史管理后台、团队协作、watchlist 或多页面导航体系
- 改造底层支付协议或 receipt 模式
