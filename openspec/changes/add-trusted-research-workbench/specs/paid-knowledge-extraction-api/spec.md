## MODIFIED Requirements

### Requirement: 支持页面触发的 Live Console
系统 MUST 提供一个页面触发的 live console，让演示者输入单条文本或白名单文章链接后，在同一页面里按阶段观察 agent session 的运行进度、支付证据和最终图谱。

#### Scenario: live console 以工作台方式展示结果
- **当** live session 成功完成
- **则** 页面主区优先展示事件总览、关键判断和证据摘录
- **且** 页面提供统一的分析凭证区域展示支付、receipt 与 payload 证据
- **且** 图谱仅作为辅助关系图保留在次级信息层

#### Scenario: live console 清晰展示导入边界与来源状态
- **当** 演示者使用白名单链接、手动文本或预置样本触发分析
- **则** 页面明确展示当前来源方式与导入状态
- **且** 当链接导入失败需改用正文输入时，页面给出清晰提示而不误导为通用链接导入能力

#### Scenario: live console 中的关键判断可回看原文证据
- **当** 页面展示某条关键判断
- **则** 用户可以看到该判断对应的原文摘录或证据锚点
- **且** 证据展示能够帮助用户复核判断是否成立

#### Scenario: live console 首屏不默认回显上一次失败或完成结果
- **当** 演示者打开 live console 页面且当前没有运行中的 live session
- **则** 页面默认展示空白工作台骨架，而不是自动恢复上一次 `completed` 或 `failed` 的 session
- **且** 只有在存在运行中的 `active` session 时，页面才自动恢复并继续轮询

#### Scenario: 主体与关系来自共享的结构化分析结果
- **当** 系统为同一篇内容生成主体与关系
- **则** `entities` 与 `relations` 应基于同一份结构化分析结果派生，而不是两次相互独立的抽取
- **且** 页面在主体抽取为空或关系不足时，应给出稳定降级表达，不把空结果包装成已完成图谱

#### Scenario: 辅助关系图在弱数据场景稳定降级
- **当** 图谱节点或关系不足以形成稳定的辅助图
- **则** 页面应降级展示主体卡片或关系清单
- **且** 当图谱数据充足时，辅助关系图应提供清晰的交互式可视化浏览体验

#### Scenario: 手动正文输入可以省略标题并自动生成展示标题
- **当** 演示者使用手动正文输入且没有填写标题
- **则** 页面默认进入手动正文输入模式，且不再提供标题输入框，只要正文有效就允许创建 live session
- **且** 当 `summary` 步骤完成后，工作台应基于该步骤的模型结果自动生成展示标题
- **且** 页面需要明确这是自动生成标题，而不是用户手动填写的原始标题
- **且** 事件总览在没有原始标题时不应直接展示自动生成标题，以避免与摘要区重复堆叠

#### Scenario: live console 首屏使用结果优先且去 demo 化文案
- **当** 用户访问 live console 页面
- **则** 页面首屏应优先展示紧凑的结果工作台与分析入口，而不是大段 demo 说明
- **且** 页面不再展示“开始演示”“比赛版”“Live Agent Console”等用户侧 demo 叙事文案

#### Scenario: 关键卡片支持弹窗查看详情
- **当** 页面展示关键判断、证据摘录或预置资讯卡片
- **则** 用户可以点击卡片打开详情弹窗查看完整内容、元信息与相关证据
- **且** 页面首屏只保留摘要式信息，避免结果区无限向下拉长

#### Scenario: graph 页面提供可缩放的交互式图谱
- **当** 用户进入独立 graph 页面且图谱数据足以成图
- **则** 页面使用 ECharts 渲染图谱，并支持缩放、拖拽和平移浏览
- **且** 页面明确说明 `derived` 连边仅用于辅助浏览，不代表真实抽取关系

#### Scenario: 强信息文本返回更高密度的主体与关系
- **当** 输入文本包含较丰富的主体、产品、事件或语义关系
- **则** 系统应尽量返回更多关键实体与关系，而不是过早截断到极小集合
- **且** 弱信息文本仍保持轻量降级策略，不把噪声堆成伪精确图谱
