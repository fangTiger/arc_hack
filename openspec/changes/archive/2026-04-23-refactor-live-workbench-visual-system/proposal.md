# 变更：重构 live 工作台视觉系统

## 为什么
当前 `live` 页面虽然已经具备工作台式的信息结构，但视觉语言仍明显延续旧版浅色、纸面化样式：
- 整体底色、字体和卡片气质仍偏“内容展示页”，不是高端科技分析终端
- 主区、侧栏、导入仓和元信息区的层级反差仍不够强，首屏焦点不够集中
- 关系图、状态条和详情层缺少统一的科技感视觉系统，页面整体记忆点不够鲜明

用户希望进入下一轮更明显的视觉重构，在不改业务逻辑的前提下，把页面改成更强的 `科技工作台 / 情报分析台` 风格。

## 变更内容
- 将 `live` 页面整体视觉系统切换为深色科技工作台风格，包括背景、字体、卡片材质、描边、辉光和状态表达
- 重排首屏视觉权重，强化 `事件总览` Hero、`关键判断` 卡组与 `证据摘录` 的阅读链路
- 收紧左侧导入仓与右侧态势栏的视觉噪声，统一详情层、关系图层和分析元信息的视觉语言
- 提供默认 `light`、可手动切换到 `dark` 的主题模式，并记住用户选择
- 为 live 页面补充浏览器页签 favicon，提升产品识别度
- 保持现有业务逻辑、页面状态模型、详情抽屉、关系图弹层和主要测试锚点不变

## 影响范围
- 受影响的规范：`paid-knowledge-extraction-api`
- 受影响的代码：`src/routes/live.ts`、相关路由测试与 E2E
