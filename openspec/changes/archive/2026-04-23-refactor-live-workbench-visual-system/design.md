# 设计：live 工作台视觉系统重构

## 设计目标
本轮只改 `视觉系统与布局表达`，不改 live session 的业务状态、轮询逻辑、详情数据结构与图谱数据来源。目标是把当前已完成的工作台结构，升级为更强的“科技工作台 / 情报分析终端”视觉表达。

## 视觉原则
1. `结果优先`：主阅读链 `事件总览 -> 关键判断 -> 证据摘录` 必须在首屏与视觉层级上最先被看到。
2. `科技感来自秩序，不来自噪声`：重点依靠深色底、冷色高光、精密边线、分区材质和密度对比，而不是廉价霓虹和过多动效。
3. `正文保持专业可读`：证据摘录、摘要、来源和凭证区保持高对比、稳定行高和较强阅读舒适度，不把“炫酷”压到正文上。
4. `测试锚点稳定`：尽量保留现有 DOM id、`data-testid` 和关键状态文案，避免让现有路由测试与 E2E 大面积失效。

## 布局策略
- 顶部状态总控条改造成低高度、强对比、横向密集的终端式状态带
- 页面主体拉宽到桌面工作台尺度，减少当前偏窄的纸面居中感
- 主区与侧栏维持现有职责，但通过材质、阴影、边线和间距拉开等级
- 左侧导入仓默认收纳为更弱的控制面板，不再保留大面积“输入页”观感
- 关系图与详情层统一成深色半透明浮层体系，视觉上与主页面同属一个终端系统
- 主题层面采用 `light` 默认、`dark` 可切换的双主题模式，确保明亮版本更适合日常阅读，深色版本保留科技工作台气质
- 主题切换状态存储在浏览器本地，避免用户每次刷新后重复切换
- 浏览器页签使用内联 favicon，提高产品识别度而不引入额外静态资源

## 实现边界
- 允许调整 `renderLiveConsolePage` 中的 CSS 与部分静态 HTML 包装结构
- 不修改 live session API、轮询协议、Graph 数据模型、工作台视图模型字段契约
- 如测试需要，仅补充与布局或稳定锚点相关的断言，不新增业务行为
