## MODIFIED Requirements
### Requirement: 支持页面触发的 Live Agent 演示台
系统 MUST 提供一个页面触发的 live console，让演示者输入单条文本或白名单文章链接后，在同一页面里按阶段观察 agent session 的运行进度、支付证据和最终图谱。

#### 场景：live console 采用深色科技工作台视觉系统
- **当** 用户访问 live console 页面
- **则** 页面整体应采用深色、高对比、科技工作台风格的视觉系统
- **且** 视觉重点应落在事件总览、关键判断和证据摘录，而不是导入区或说明性文案
- **且** 页面不得继续呈现明显的浅色纸面风、演示页或内容站排版气质

#### 场景：主区与侧栏形成明确的视觉等级
- **当** 页面首屏展示工作台内容
- **则** 主区应通过体量、材质和对比度成为绝对主视觉
- **且** 左侧导入仓与右侧态势栏只能作为辅助层，不得与主区形成同权重竞争
- **且** 分析凭证、来源与流程等元信息应在视觉上弱于主结论与证据阅读区

#### 场景：关系图与详情层共享终端式浮层视觉
- **当** 用户展开辅助关系图或打开详情抽屉
- **则** 浮层应与主工作台共享统一的深色终端视觉语言
- **且** 关系图弹层、详情抽屉和分析元信息面板应保持一致的材质、描边与层次系统
- **且** 页面不应同时混用浅色纸面卡片和独立的演示型弹窗风格

#### 场景：页面默认提供 light 主题并支持手动切换
- **当** 用户首次访问 live console 页面
- **则** 页面默认应以 `light` 主题展示，避免首屏过暗影响阅读
- **且** 页面应提供明确的主题切换入口，让用户可手动切换到 `dark` 主题
- **且** 用户刷新页面后，系统应继续沿用上一次手动选择的主题

#### 场景：浏览器页签展示工作台图标
- **当** 用户在浏览器中打开 live console 页面
- **则** 页面应提供 favicon 或等价页签图标
- **且** 图标应与工作台产品身份保持一致，而不是浏览器默认空白图标
