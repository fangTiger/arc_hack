## MODIFIED Requirements
### Requirement: 支持页面触发的 Live Agent 演示台
系统 MUST 提供一个页面触发的 live console，让演示者输入单条文本或白名单文章链接后，在同一页面里按阶段观察 agent session 的运行进度、支付证据和最终图谱。

#### 场景：live console 具备明确的 Arc 品牌锚点
- **当** 用户访问 live console 页面
- **则** 页面顶部应展示清晰的 Arc 品牌元素，包括 Arc 风格图标或 wordmark
- **且** 浏览器标签页应使用 Arc 风格 favicon，而不是通用默认图标
- **且** Arc 品牌元素应与 Arc / Circle / USDC / Gateway 的工作台语义保持一致，而不是独立广告位

#### 场景：预置资讯优先从动态 Arc 相关来源生成
- **当** 用户访问 live console 页面
- **则** 预置资讯区应优先展示由服务端动态生成的 Arc 相关资讯卡片
- **且** 动态卡片只能来自当前支持解析的白名单来源
- **且** 当动态来源抓取不足或失败时，页面必须自动回退到静态预置卡片，保持入口可用

#### 场景：页面提供可直接试跑的真实链接入口
- **当** 用户准备验证链接导入能力
- **则** 页面应提供至少一个已验证可解析的真实链接入口或填充动作
- **且** 该入口应帮助用户快速试跑，而不是要求用户自行寻找可用来源
